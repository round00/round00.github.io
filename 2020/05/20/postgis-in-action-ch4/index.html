PostGIS实战-第四章 几何函数

May 20 2020 PostGIS>PostGIS In Action 25 minutes read (About 3682 words)

PostGIS实战一系列的文章暂时不写了，原因看下文章末尾。主要就是版本变动太大了。

前面的章节中，我们见到了PostGIS提供的各种几何对象以及如何创建它们。在本章以及后面的章节中，我们会介绍一系列核心的函数来操作这些几何对象。本章会专注于处理单个几何对象的函数。

PostGIS提供了300+的函数和操作符。为了能方便的概览所有函数，我们开发了一种根据使用方法来区别的分类法。这并不是很严谨的分类。按照完成任务的类型来将函数分类，是最方便有效的办法了。下面我们浏览一下分类目录：

  • 构造函数：从WKT或者WKB来创建几何对象

  • 输出函数：将几何对象以一些知名的标准格式（例如：WKT、WKB、GML、SVG、KML、GeoJson等）输出

  • 读取/设置函数：用于操作单独的几何对象，返回/设置它的属性

  • 分解函数：从输入的几何对象中导出几何对象。

  • 组合函数：把几何对象拼接、组合到一起

  • 度量函数：返回几何对象的某个度量标量值

  • 简化函数：有时候并不需要一个几何对象的完全体，这些函数通过删除点、线串或者近似舍入坐标来简化几何对象。产生的结果几何对象会和原来的看起来基本一样。只是精度低了。

按照本书的使命：展示如何使用PostGIS，而不是作为一个查阅的文档，这里只会介绍一些常用的函数，如果需要完整的函数列表，要去官方文档找。

函数的前缀ST是什么意思？

可以注意到，基本上所有函数都是以ST开头的。S表示spatial（空间的），T表示temporal（时间的）。ST前缀在其他空间数据库中常常被省略掉，但是在PostGIS中留下来了。

一、构造函数

构造函数用于创建几何对象，有两种常用的创建几何对象的方式。第一种是使用可以接受的原始数据格式来创建，第二种是从其他已经存在的几何对象通过分解、拼接、切片、索引来生成新的。

1、从WKT或WKB格式创建

这些不可或缺的函数在你输入各种文本或者二进制数据时，会输出几何对象。这些格式可以在其他桌面工具中能方便的看到几何对象。

注：WKT（well-known text），WKB（well-known binary）

1. ST_GeomFromText

这个函数时SQL/MM标准函数，在其他兼容SQL/MM标准的空间数据库中，也可以找到这个函数。根据SQL/MM规定，这个函数只支持2D。

2. ST_GeomFromEWKT

这是另一个类似的函数，只在PostGIS中有，因为EWKT（extended WKT）是PostGIS独有的格式，用于弥补WKT的不足之处。EWKT将SRID直接编码到WKT中，并且支持3D、4D。EWKT显式的前置SRID到WKT中。

3. ST_GeomFromWKB 和 ST_GeomFromEWKB

同上，ST_GeomFromWKT是SQL/MM标准的函数。ST_GeomFromEWKT是PostGIS扩展出的函数，用于编码SRID，支持3D、4D。这两个函数接收字节流数组，而不是字符串。字节流的一个优点是精确。比如用字符串的话，在小数点15位之后就会被截断。

二、输出函数

输出函数是返回几何对象的其他工业标准格式表示的函数。这样的表示形式能被其他第三方工具来使用。

本节我们会总结可用的输出格式，它们的通用场景，以及PostGIS如何输出那种格式。先要了解某种格式的具体细节的话，需要去它的官方文档查看，这里不讨论它们的细节。

1、WKT和WKB

对于几何对象来说，WKT是最通用的OGC标准格式。

几个输出函数有：ST_AsText、ST_AsEWKT、ST_AsBinary和ST_AsEWKB。

这几个函数是上一节中几个构造函数的相反的函数。

2、KML

KML全称是Keyhole Markup Language，是一种基于XML的语言，是OGC标准支持的标准传输格式。KML的空间参考系是4326。即使你的几何对象是一个其他的SRID，KML也会自动的帮你转换为4326。

输出KML格式的函数为：ST_AsKML。它支持2D和3D，但是对于曲线几何和容器会报错。KML不会输出M维。

3、GML

GML全称是Geography Markup Language，是一种基于XML的语言，是OGC标准支持的标准传输格式。

输出GML格式的函数为：ST_AsGML。对于几何类型和容器都支持2D、3D，M维会被忽略。曲线几何会报错。

4、GeoJSON

GeoJSON的全称是Geometry JavaScript Object Notation，是一种基于JSON的格式。

输出GeoJSON格式的函数为：ST_AsGeoJSON。对于几何类型和容器都支持2D、3D，M维会被忽略。曲线几何会报错。

5、SVG

SVG的全称是Scalable Vector Graphics，可缩放矢量图形。

输出SVG格式的函数为：ST_AsSVG。仅支持2D。

6、Geohash

Geohash是一种用于经纬度的有损地理编码系统。

输出Geohash的函数为：ST_AsGeohash。它永远输出4326参考系的坐标。它可以支持曲面几何，但是会忽略Z和M维。

三、读取/设置函数

如果你用过一种面向对象语言，那么一定对getters和setters不会陌生，这一节的函数就和它们一样。这一系列的函数就是来自于面向对象编程，它们用来获取或设置几何对象的内在属性。

1、空间参考系

空间参考系允许有意义的测量，并且使得分享数据可能实现。

空间参考系的获取设置函数为：ST_SRID和ST_setSRID。它们都是SQL/MM标准。

记住每一个几何对象都必须有一个SRID，即使是一个未知的SRID（-1）。

3、几何类型

在大部分情况下，你清楚的知道你正在操作的几何类型，但是有的时候你并不清楚。这时你可以借助两个函数：GeometryType和ST_GeometryType。前者已经被标记为要废弃的了。这两者不仅互不相同，而且它们也都很有用。

前者出现的时间比较早，它是OGC标准的。它返回的类型是我们所熟悉的，全部大写的。例如POINT、POLYGON。

后者出现较晚，它是OpenGIS SQL/MM的一部分。它返回以ST为前缀的类型。例如ST_Point、ST_Polygon。

4、坐标和几何维度

这里注意个事情：坐标维度和几何维度的区别。

  • 坐标维度是几何对象所在的向量空间的维度，在PostGIS中可以是2、3、4。

  • 几何维度是为了完全盛下几何对象所需要的最小向量空间维度。在PostGIS中可以是：0（点）、1（线串）、2（多边形）。

PostGIS提供了ST_CoordDim和ST_Dimension来分别返回坐标维度和几何维度。

5、几何图形有效性

非法的几何图形，例如自相交或者内环在外环之外的多边形。可以说，几何维度越高，它就越可能是非法的。

ST_isValid可以验证几何图形的有效性，ST_IsValidReason可以查看几何图形非法的简单描述，有多条原因也只返回一条。ST_IsValidDetail返回一个非法原因的集合，

不要尝试操作一个非法的几何对象。许多基于GEOS的函数遇到非法的几何对象可能会有预料不到的结果。

6、定义几何对象的点的个数

ST_NPoints可以返回几何对象中点的个数，它可以应用于任意几何对象。还有另外一个函数ST_NumPoints作用和它差不多。但是后者只能用于线串。当应用于多线串时，只会返回第一个线串的点数。

四、度量函数

平面测量通常以米或英尺为单位。平面模型再GIS工具中被支持的更好一些，处理的也更快一点。如果只是在大陆上进行测量的话，平面通常也是首选，虽然会浪费一些精度。当跨越大陆或者海洋时，平面模型的效果开始急速的变坏。这时就要用地理模型了，必须考虑地球的球形特性。SRID4326是最常用的地理空间参考系。

2、几何类型的地理测量

之前讨论的都是在笛卡尔系中的几何测量，现在开始看下球坐标系的测量。大地测量是球测量的一个更华丽的术语。例如在地球上位处两个对面的地点，如果直线相连它们，这条线会穿过地心。而在地面上，有无数条曲线可以连接它们，尽管你总是挑一条最短的曲线，但是也没办法保证它唯一。
